# CLAUDE.md

This project uses a three-doc knowledge workflow. **At session start, read `PROJECT.md`, `NOTES.md`, and `DEVELOPMENT_STATE.md` before doing anything else** — together they cover the slow-moving spec, the medium-velocity thinking still in flight, and the per-session handoff. Each file has a header that says what role it plays and how it relates to the others.

At the end of any session that produced substantive work, propose an update to `DEVELOPMENT_STATE.md` (a new session entry) and any `NOTES.md` threads the work touched. If a thread in `NOTES.md` has settled into doctrine, suggest graduating it to `PROJECT.md` rather than leaving settled material in the notebook.
